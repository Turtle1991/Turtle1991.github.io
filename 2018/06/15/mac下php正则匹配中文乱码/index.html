mac下php正则匹配中文乱码

码日子
  • php

这个问题遇到了两次，最近一次是一个页面在返回数据的时候json_encode()返回false，但是测试环境又是正常的，只有本地出现，猜想是否是数据中有乱码出现导致，在本地把数据打印出来，发现数据中有部分中文出现了乱码，此时猜测是不是数据库数据编码和本地不一致？但是检查后发现，数据库、本地代码都是UTF-8编码。排除数据源问题，那估计就是在数据处理过程中哪里出现问题，往前一步步调试，最后发现在一个正则处理后出现乱码。

此正则是用于替换掉重复的空白符，看到这里，想起之前好像处理过类似的问题，猜测是不是和系统有关，于是开始问度娘，最后发现增加模式修正符u就可以了。

根据资料，是这么说：

u（PCRE_UTF8）

此修正符启用了一个 PCRE 中与 Perl 不兼容的额外功能。模式字符串被当成 UTF-8。本修正符在 Unix 下自 PHP 4.1.0 起可用，在 win32 下自 PHP 4.2.3 起可用。自 PHP 4.3.5 起开始检查模式的 UTF-8 合法性。

或许是因为平台差异，在mac下字符串匹配出现了问题，而centos这类系统正常。所以加上修正符使得字符串按UTF-8格式去匹配。
