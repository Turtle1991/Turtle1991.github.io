mac下php开发环境搭建

码日子
  • mac
  • 配环境

准备工作

1.安装好 Xcode:
从app store中搜索安装，然后命令行 xcode-select --install ，输入后会提示安装命令行开发工具，安装完后执行下面的操作

cd /Applications/Xcode.app/Contents/Developer/Toolchains/
sudo ln -s XcodeDefault.xctoolchain OSX10.10.xctoolchain

2.安装 homebrew:

ruby -e "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)”

有时候会因为网络问题安装失败 提示 curl Server aborted the SSL handshake 啥的 那就先执行下面的命令进行卸载 然后在来一次安装命令 中间会提示输入系统的密码 看到success 就是成功了。

ruby -e "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/uninstall)”

使用：
安装： brew install xxx
删除： brew remove xxx
查看： brew list
更新： brew upgrade
更新brew： brew update

安装apache

安装方式和centos下一样，参考[LAMP环境搭建]Apache安装，我是把环境相关的东西都安装在一个指定的目录里（如 /usr/local/turtle/），这样以后可以方便管理。

mkdir /usr/local/turtle
chown Turtle:staff /usr/local/turtle //更改权限
mkdir /usr/local/turtle/src //来存放解压的安装包

安装完后启动时 会报 cannot bind address什么的，我觉得可能是权限问题，因为我启动时是以本地用户Turtle启动，所以觉得是权限被限制，换成以root启动即可 。

sudo ./apachectl start

安装mysql

一些想法

1.本来想添加一个mysql组的，跟之前centos安装mysql一样，但是在网上查后发现说 mac系统已经有一个_mysql组了，且这个组有个别名就叫 mysql，所以如果自己再加一个可能会有问题，而且觉得没有必要。

mac下关于组的一些命令：

dscl . -read /Groups/mysql  //查看组的信息
dscl . -create /Groups/mysql gid 296 //创建组 并指定一个组id
dscl . -passwd /Users/mysql ‘my secret’ //设置密码

2.原先打算使用dmg包来安装，直接下载后点击安装即可，默认没有配置文件，所以要cp /usr/local/mysql/support-files/my-default.cnf /etc/my.cnf，然后按照centos下mysql的配置就可以了，但是觉得这样安装后有的文件不好管理，还是喜欢用编译安装到指定的位置，所以不采用此方法。

安装

采用tar包安装，到官网下载后解压

tar
cp 解压后的目录 /usr/local/mysql
sudo chown -R root:wheel /usr/local/mysql //更改权限
mkdir /usr/local/mysql/data
sudo chown _mysql:_mysql /usr/local/mysql/data //更改权限
//复制配置文件 配置安装centos下mysql的配置就可以
cp /usr/local/mysql/support-files/my-default.cnf /etc/my.cnf
//初始化
/usr/local/mysql/bin/mysqld —initialize —user=_mysql --basedir=/usr/local/mysql --datadir=/usr/local/mysql/data
/usr/local/mysql/support-files/mysql.server start //启动
/usr/local/mysql/support-files/mysql.server stop //停止
/usr/local/mysql/support-files/mysql.server restart //重启

创建软链接，PHP编译安装需要用到，不然后面PHP编译的时候会是提示关于mysqlclient的错误(Cannot find libmysqlclient_r under /usr/local/mysql/)

sudo ln -s /usr/local/mysql/lib/libmysqlclient.20.dylib /usr/lib/libmysqlclient.20.dylib

可能遇到的问题

1.开启后使用 mysql -uroot -p 无法登入
解决方法：

先关闭mysql服务
sudo su //切换到root用户
/usr/local/mysql/bin/mysqld_safe --skip-grant-tables --skip-networking &

这样便可以越过权限表，跳过密码认证过程 登入。新开一个终端 mysql -uroot -p 不用输入密码即可登入mysql。看这个版本的mysql(5.10) mysql.user 表中没有了 password 字段，替换的字段是 authentication_string，所以修改密码则更新此字段。

mysql> update user set authentication_string=PASSWORD('密码') where User='root’;
mysql> flush privileges;

退出后重新登入，可以进 可是当我执行 show databases;的时候提示

mysql> show databases;
ERROR 1820 (HY000): You must reset your password using ALTER USER statement before executing this statement.

解决方法：

mysql> set password=password('密码');
Query OK, 0 rows affected, 1 warning (0.00 sec)

mysql> alter user 'root'@'localhost' password expire never;
Query OK, 0 rows affected (0.00 sec)

mysql> flush privileges;
Query OK, 0 rows affected (0.00 sec)

mysql> exit;

重新登入就可以了。

安装PHP

准备工作

1.此部分安装可以参考[LAMP环境搭建]php安装

2.编译安装 libXpm
要先安装x11 和 一个什么 config的，什么config这个用brew安装就可以了，x11看系统里面有，点击后进入官网下载直接安装即可。

3.编译安装 libiconv
之所以要自己编译安装是因为不安装的话，下面php make时候总是会报错，好像是自带的iconv版本太低了。

tar 
cd 
./configure /usr/local/turtle/libiconv 
make 
make install

安装PHP

tar
cd
./configure --prefix=/usr/local/turtle/php5.6.16 --with-iconv=/usr/local/turtle/libiconv/ --with-apxs2=/usr/local/turtle/apache2418/bin/apxs --with-config-file-path=/usr/local/turtle/php5.6.16/etc --with-libxml-dir=/usr/local/turtle/libxml2/ --with-zlib-dir=/usr/local/turtle/zlib/ --with-gd=/usr/local/turtle/gd/ --with-jpeg-dir=/usr/local/turtle/jpeg9/ --with-png-dir=/usr/local/turtle/libpng/ --with-freetype-dir=/usr/local/turtle/freetype/ --with-mysql --with-mysqli --with-openssl --enable-gd-native-ttf --enable-mbstring --enable-ftp --enable-bcmath --enable-sockets --enable-zip --enable-soap --enable-calendar --with-curl --with-pdo-mysql --enable-xml --enable-maintainer-zts --without-pear --with-xpm-dir=/usr/local/turtle/libXpm/
make
make install

拓展安装

brew install libmcrypt
brew install autoconf

可能遇到的问题

1.安装 libmemcached make的时候报错

make[1]: *** [libmemcached/libmemcached_libmemcached_la-byteorder.lo] Error 1
make: *** [all] Error 2

在stackoverflow上找到了解决方法：

I’ve managed to get libmemcached to compile and install on OSX 10.10.3 by changing the configure script and one of the source files.
In configure I changed the lines:

to:
ac_cv_have_htonll=no
Then in clients/memflush.cc I changed the 2 instances of:
if (opt_servers == false)
to:
if (opt_servers == NULL)
After that the usual configure, make, sudo make install worked ok (albeit with lots of warnings). The library then seems to work fine, with pylibmc in my case.
