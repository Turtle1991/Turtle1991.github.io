[LAMP环境搭建] php安装

码日子
  • php
  • 配环境

先编译安装需要的包

基本

# yum install curl-devel
# yum -y install python-devel (ubuntu 要安装 apt-get install python-dev)

安装 libxml2

wget ftp://xmlsoft.org/libxml2/libxml2-2.9.1.tar.gz
tar
cd
./configure --prefix=/usr/local/libxml2

安装 zlib

wget  http://zlib.net/zlib-1.2.8.tar.gz
tar
cd
./configure --prefix=/usr/local/zlib

安装 libpng

wget http://sourceforge.net/projects/libpng/files/libpng16/1.6.10/libpng-1.6.10.tar.gz/download
tar 
cd 
cp scripts/makefile.linux makefile 
make 
make install

注意: 这里的 makefile 不是用 ./configure 生成的 , 而是直接从 script/ 目录里拷一个,这一步结束时， libpng 被安装在 /usr/local/lib 下，我采用的方案是指定好安装的目录，不然下面安装gd库会找不到文件

./configure --prefix=/usr/local/libpng

安装 freetype:

wget  http://download.savannah.gnu.org/releases/freetype/freetype-2.5.3.tar.gz
tar
cd
./configure --prefix=/usr/local/freetype 
make 
make install

安装 jpeg6

建立目录  
mkdir /usr/local/jpeg6 
mkdir /usr/local/jpeg6/bin 
mkdir /usr/local/jpeg6/lib 
mkdir /usr/local/jpeg6/include 
mkdir /usr/local/jpeg6/man 
mkdir /usr/local/jpeg6/man/man1 
wget http://www.ijg.org/files/jpegsrc.v6b.tar.gz
tar -zxvf jpegsrc.v6b.tar.gz 
cd jpeg-6b 
./configure --prefix=/usr/local/jpeg6/ --enable-shared --enable-static 
make 
make install

如果make出错： (使用新版本的 就不会报这个错了)
./libtool –mode=compile gcc -O2 -I. -c ./jcapimin.c
make: ./libtool：命令未找到
make: * [jcapimin.lo] 错误 127
则进入jpeg-6b的源码目录，也就是把 libtool里面的两个配置文件拿来覆盖掉jpeg-6b目录下的对应文件，make clean 再重新configure： 

[ root@bogon jpeg-6b]# cp /usr/share/libtool/config/config.sub . 
[ root@bogon jpeg-6b]# cp /usr/share/libtool/config/config.guess .

安装gd库 2.1.0

先安装libxpm，让gd库支持xpm，不然后续编译PHP会报错。

yum install libXpm-devel.x86_64 (apt-get install libxpm-dev)

wget https://bitbucket.org/libgd/gd-libgd/downloads/libgd-2.1.0.tar.gz
tar
cd
./configure --prefix=/usr/local/gd --with-png=/usr/local/libpng --with-freetype=/usr/local/freetype --with-jpeg=/usr/local/jpeg6
make
make install

编译安装PHP

获取PHP源码包

wget
tar
cd
./configure --prefix=/usr/local/php --with-apxs2=/usr/local/apache/bin/apxs --with-config-file-path=/usr/local/php/etc --with-libxml-dir=/usr/local/libxml2 --with-zlib-dir=/usr/local/zlib --with-gd=/usr/local/gd --with-jpeg-dir=/usr/local/jpeg6 --with-png-dir=/usr/local/libpng --with-freetype-dir=/usr/local/freetype --with-mysql=/usr/local/mysql --with-mysqli=/usr/local/mysql/bin/mysql_config --with-openssl --enable-gd-native-ttf --enable-mbstring --enable-ftp --enable-bcmath --enable-sockets --enable-zip --enable-soap --enable-calendar --with-curl --with-pdo-mysql --enable-xml --with-iconv --enable-maintainer-zts --without-pear
(若编译出现问题，查看下面问题解决列表)
make
make install

如果要为fpm模式，则加上 –enable-fpm，此部分后续整理。

可能遇到的问题

问题一:

checking for cURL in default path… not found
configure: error: Please reinstall the libcurl
distribution -
easy.h should be in /include/curl/

解决: yum -y install curl-devel

问题二:

/usr/local/src/php-5.5.10/ext/gd/gd.c:57:22: error: X11/xpm.h: No such file or directory
make: * [ext/gd/gd.lo] Error 1
问题说明：这是提示gd库不支持xpm

解决方法：
1.安装libxpm

yum install libXpm-devel.x86_64 （apt-get install libxpm-dev）

2.然后重新安装gd库

问题三:

如果编译过程中遇到下列信息
PEAR package PHP_Archive not installed: generated phar will require PHP¨s phar extension be enabled.
这说明没有安装PEAR，

解决方案有两种：
1、./configure时附加–without-pear
2、下载未打包的pear安装包并执行，步骤如下： 

# wget  http://pear.php.net/go-pear.phar  
# /usr/local/webserver/php/bin/php go-pear.phar

我选了第一种方案

安装完配置

复制配置文件

[ root@bogon php-5.5.10]# mkdir /usr/local/php/etc 
[ root@bogon php-5.5.10]# cp php.ini-production /usr/local/php/etc/php.ini

修改apache配置文件

vim /usr/local/apache/conf/httpd.conf
可以修改DocumentRoot目录 

在这写入 
<IfModule dir_module>
     DirectoryIndex index.php index.html 
</IfModule> 
在<IfModule mime_module>中添加： 
    AddType application/x-httpd-php .php  
    AddType application/x-httpd-php-source .phps 
    AddDefaultCharset utf8 (这句使 apache 默认支持 utf8 字符集)

重启apache service apache restart

PS:
编辑 php.ini 文件，找到;default_charset="iso-8859-1"，增加一行： default_charset="utf8"，这个可以加也可以不加。

然后就可以测试PHP了 curl localhost
