[LAMP环境搭建] Apache安装

码日子
  • apache
  • 配环境

删除自带的apache

service httpd stop        
yum remove httpd

安装一些包

yum -y install gcc gcc-c++ libtool

下载源码包并安装

安装步骤：
wget
tar -zxvf
cd
./configure
make
make install

apr

wget http://mirrors.hust.edu.cn/apache//apr/apr-1.5.0.tar.gz
tar -zxvf  
cd        
./configure --prefix=/usr/local/apr
make
make install

apr-util

wget http://mirrors.hust.edu.cn/apache//apr/apr-util-1.5.3.tar.gz  
tar -zxvf
cd
./configure --prefix=/usr/local/apr-util --with-apr=/usr/local/apr
make
make install

pcre

wget ftp://ftp.csx.cam.ac.uk/pub/software/programming/pcre/pcre-8.34.tar.gz
tar -zxvf
cd
./configure --prefix=/usr/local/pcre
make
make install

apache 

wget http://mirrors.cnnic.cn/apache//httpd/httpd-2.4.9.tar.gz
tar -zxvf 
cd
./configure --prefix=/usr/local/apache --enable-so --enable-cgi --enable-ssl --enable-rewrite --enable-mods-shared=most --enable-module=so --enable-proxy --with-apr=/usr/local/apr/ --with-apr-util=/usr/local/apr-util/ --with-pcre=/usr/local/pcre
make
make install

如果make不成功，那么是configure不成，例如你缺少ssl组件，可以去掉–enable-ssl

安装成功后

修改配置文件
里面的servername 改为 localhost:80
#LoadModule slotmem_shm_module modules/mod_slotmem_shm.so
去掉注释 载入模块 (不然的话启动后会没有反应，查看错误日志后知道是这个模块没有开启)
/usr/local/apache/bin/apachectl -t 检查配置是否有误
启动 /usr/local/apache/bin/apachectl start
lsof -i:80 查看是否开启,也可以本机访问localhost:80

将apache加入系统服务 随机启动,首先以apachectl脚本为模板生成Apache服务控制脚本：

grep -v "#" /usr/local/apache-2.2.15/bin/apachectl  > /etc/init.d/apache
vim /etc/rc.d/init.d/apache
在文件最前面插入下面的行，使其支持chkconfig命令：
#!/bin/sh 
# chkconfig: 2345 85 15 
# description: Apache is a World Wide Web server. 
保存后退出vi编辑器，执行下面的命令增加Apache服务控制脚本执行权限：
chmod  +x  /etc/init.d/apache 
执行下面的命令将Apache服务加入到系统服务：
chkconfig --add apache
执行下面的命令检查Apache服务是否已经生效：
chkconfig --list apache
命令输出类似下面的结果：
apache          0:off 1:off 2:on 3:on 4:on 5:on 6:off
表明apache服务已经生效，在2、3、4、5运行级别随系统启动而自动启动

以后可以使用service命令控制Apache的启动和停止

启动Apache服务：service apache start 
停止Apache服务：service apache stop  
关闭开机自启动： chkconfig apache off 
改变开机自启动的运行级别为3、5：chkconfig --level 35 apache on

可能遇到的问题

有时候不能访问是由于防火墙的问题，可以关闭掉防火墙(我选择关闭防火墙，虚拟机下防火墙没啥用)，或者修改配置，允许80端口访问:

vim /etc/sysconfig/iptables    

-A INPUT -m state --state NEW -m tcp -p tcp --dport 22 -j ACCEPT
-A INPUT -m state --state NEW -m tcp -p tcp --dport 80 -j ACCEPT
-A INPUT -j REJECT --reject-with icmp-host-prohibited

改后重启防火墙 /etc/init.d/iptables restart

其他

开启URL重写

1.打开httpd.conf
2.找到LoadModule rewrite_module modules/mod_rewrite.so并将其前面的#去掉
3.找到Directory配置节点 ，将AllowOverride None 改成 AllowOverride All
4.重启APACHE

apache多站点配置

将apache配置文件中(发现 http.conf 这个文件的 DocumentRoot 也要设置，要设置的各个虚拟主机的目录必须是归属于这个目录，像子目录一样)
#LoadModule vhost_alias_module modules/mod_vhost_alias.so(启用apache的虚拟主机功能)
#Include conf/extra/httpd-vhosts.conf
(从conf/extra/httpd-vhosts.conf导入虚拟主机配置)
注释去掉

编辑配置文件 

vim /usr/local/apache/conf/extra/httpd-vhosts.conf

<VirtualHost *:80> 
    ServerAdmin  test@test.com
    DocumentRoot "/path/to/one/web" 
    ServerName test.com 
    ServerAlias  www.test.com
    ErrorLog "logs/test-error_log" 
    CustomLog "logs/test-access_log" common 
</VirtualHost> 
<Directory "/path/to/one/web">
    Options Indexes FollowSymLinks 
    AllowOverride All 
    Order Deny,Allow 
    Allow from all 
</Directory>

PS

如果是安装在虚拟机中，要让虚拟机可以访问我们设置的站点，需要修改虚拟机的host，要让主机访问虚拟机，同理修改主机的host文件

虚拟机：
vim /etc/hosts 
加入 
127.0.0.1    test.com
主机：
虚拟机ip    test.com

重启apache，测试
